# Security Policy

## Supported Versions

The following versions of the **Driving School Management System** are currently supported with security updates.

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅         |
| < 1.0   | ❌         |

> **Note:** This project is maintained as an academic project. Security updates will be provided only for the latest stable version.

---

## Reporting a Vulnerability

The Driving School Management System Project Team takes security seriously and appreciates responsible disclosure of potential vulnerabilities.

If you discover a security issue, please do **not** disclose it publicly. Instead, report it privately to the project team by creating a private communication through one of the following methods:

* Contact the project maintainers via email.
* Open a private security advisory on GitHub (if enabled).
* If private reporting is unavailable, contact one of the project team members directly.

When reporting a vulnerability, please include:

* A clear description of the issue.
* Steps to reproduce the vulnerability.
* The affected version.
* Any screenshots, logs, or proof-of-concept code (if applicable).
* Suggested mitigation or fix (optional).

---

## Response Process

After receiving a vulnerability report, the project team will:

1. Acknowledge receipt of the report within **3–5 business days**.
2. Investigate and verify the reported issue.
3. Notify the reporter whether the vulnerability has been accepted or declined.
4. Develop and test an appropriate fix if the issue is confirmed.
5. Release a security update and publicly acknowledge the reporter (with permission).

---

## Scope

This security policy applies to:

* Source code
* Authentication and authorization
* User account management
* Database interactions
* File uploads
* API endpoints
* Third-party dependencies

---

## Responsible Disclosure

Please allow the project team sufficient time to investigate and resolve reported security issues before making any public disclosure. Responsible disclosure helps protect users and improves the security of the project.

Thank you for helping keep the **Driving School Management System** secure.
